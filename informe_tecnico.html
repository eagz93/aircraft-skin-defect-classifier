1. Introducción

1.1 Motivación y contexto del problema

La inspección de superficies de aeronaves es un proceso crítico para garantizar la aeronavegabilidad y seguridad operacional. Tradicionalmente, esta inspección se realiza de forma visual por técnicos certificados, lo que conlleva limitaciones inherentes: subjetividad del operador, fatiga durante inspecciones prolongadas, variabilidad inter-observador y tiempos de inactividad significativos de la aeronave.

Los defectos superficiales en aeronaves — grietas (cracks), abolladuras (dents), rayones (scratches), desprendimiento de pintura (paint-off) y cabezas de remaches faltantes (missing heads) — presentan patrones visuales distintivos que los hacen candidatos ideales para clasificación automatizada mediante técnicas de visión por computadora y aprendizaje profundo.

Este proyecto desarrolla progresivamente un sistema de clasificación de defectos en superficies de aeronaves, avanzando desde modelos simples (MLP) hasta arquitecturas estado del arte (Vision Transformers con LoRA), integrando generación de datos sintéticos y desplegando el sistema como aplicación web funcional.

1.2 Objetivos del proyecto

  • Objetivo general: Diseñar, implementar y desplegar un sistema de deep learning para la clasificación automatizada de 5 tipos de defectos en superficies de aeronaves.
  • Objetivos específicos:
    • Evaluar progresivamente arquitecturas de complejidad creciente: MLP → CNN → Transfer Learning → ViT + LoRA.
    • Implementar un componente generativo (VAE + DCGAN) para aumentar y balancear el dataset.
    • Optimizar el modelo final mediante LoRA/PEFT, cuantización y pruning.
    • Desplegar una interfaz web interactiva con clasificación, localización y explicabilidad.

2. Descripción del Dataset

2.1 Fuentes de datos

Se combinaron dos datasets públicos de la plataforma Roboflow Universe, ambos con licencia CC BY 4.0:

Dataset Autor Imágenes aprox. Tipo
aircraft-skin-defects-merged-final Dibya Dillip ~2,000 Object Detection (COCO)
aircraft-skin-defects AI Assistant for General Visual Inspection ~4,600 Object Detection (COCO)

Ambos datasets contienen anotaciones en formato COCO con bounding boxes para 5 clases unificadas de defectos superficiales.

2.2 Análisis exploratorio

Tras la descarga, unificación y recorte de parches (crops) de las regiones anotadas, se obtuvo un corpus de 33,901 imágenes distribuidas en 5 clases:

Clase Descripción Muestras Proporción
crack Grietas en la superficie metálica 8,088 23.9%
dent Abolladuras por impacto 10,064 29.7%
missing_head Cabezas de remaches faltantes 7,510 22.1%
paint_off Desprendimiento de pintura 6,730 19.9%
scratch Rayones superficiales 1,509 4.5%
Desbalance de clases: La clase scratch representa solo el 4.5% del total (1,509 muestras), mientras que dent alcanza el 29.7% (10,064 muestras) — una proporción de 6.7:1. Este desbalance motivó la generación de datos sintéticos en la Etapa 4.

2.3 Preprocesamiento y partición

El pipeline de preprocesamiento incluye:

  1. Extracción de parches: Recorte de regiones de interés (ROI) a partir de bounding boxes COCO.
  2. Redimensionamiento: 128×128 px para MLP (escala de grises, aplanado a vector 1D de 16,384 dimensiones), 224×224 px para CNN, ResNet50 y ViT.
  3. Normalización: Media y desviación estándar de ImageNet para modelos preentrenados; normalización [0,1] para modelos from-scratch.
  4. Partición estratificada: 70% entrenamiento / 15% validación / 15% test (semilla fija = 42).

3.1 Etapa 1 — MLP Base (Perceptrón Multicapa)

Se implementó un MLP clásico como modelo base de referencia. Las imágenes se convirtieron a escala de grises, se redimensionaron a 128×128 px y se aplanaron en vectores de 16,384 dimensiones.

Arquitectura

Input(16,384) → Linear(512) → BN → ReLU → Dropout(0.3)
             → Linear(256) → BN → ReLU → Dropout(0.3)
             → Linear(128) → BN → ReLU → Dropout(0.3)
             → Linear(5) → Softmax

Hiperparámetros: lr = 1×10⁻³, weight_decay = 1×10⁻⁴, 30 épocas, early stopping (patience = 7), optimizador Adam, loss CrossEntropy con pesos de clase.

Justificación: El MLP sirve como línea base deliberadamente limitada — al aplanar las imágenes pierde toda la estructura espacial, lo que justifica la evolución hacia arquitecturas convolucionales.

3.2 Etapa 2 — CNN Profunda

Se diseñó una CNN modular desde cero con bloques convolucionales apilables para capturar la estructura espacial de las imágenes.

Arquitectura (configuración óptima: 6 bloques)

Input(3, 224, 224)
  → [Conv3×3(64) → BN → ReLU → MaxPool2×2]    # Bloque 1
  → [Conv3×3(128) → BN → ReLU → MaxPool2×2]   # Bloque 2
  → [Conv3×3(256) → BN → ReLU → MaxPool2×2]   # Bloque 3
  → [Conv3×3(512) → BN → ReLU → MaxPool2×2]   # Bloque 4
  → [Conv3×3(512) → BN → ReLU → MaxPool2×2]   # Bloque 5
  → [Conv3×3(512) → BN → ReLU → AdaptiveAvgPool]  # Bloque 6
  → Flatten → Linear(512) → Dropout(0.5) → Linear(5)

Estudio de ablación: Se evaluaron 6 configuraciones variando profundidad (4 vs 6 bloques), dropout (0.3, 0.5, 0.7) y tasa de aprendizaje (10⁻⁴, 10⁻³, 10⁻²) para identificar la combinación óptima.

3.3 Etapa 3 — Modelos Preentrenados y Transfer Learning

Se integraron dos arquitecturas preentrenadas en ImageNet para aprovechar representaciones visuales generales:

ResNet50

Red residual de 50 capas con conexiones skip. Entrenamiento en dos fases: (1) backbone congelado, solo cabeza clasificadora — 10 épocas; (2) fine-tuning de las últimas 2 capas residuales — 20 épocas con lr reducido (1×10⁻⁴).

ViT-Base (Vision Transformer)

Modelo google/vit-base-patch16-224 de Hugging Face. Divide la imagen en parches de 16×16 y aplica atención multi-cabeza (self-attention). Entrenamiento en dos fases análogo a ResNet50: backbone congelado → fine-tuning parcial de los últimos 3 bloques transformer.

Nota: El ViT-Base fue seleccionado como modelo base para la Etapa 5 (LoRA) por su arquitectura modular que facilita la inyección de adaptadores de bajo rango.

3.4 Etapa 4 — Componente Generativo (VAE + DCGAN)

Se implementaron dos arquitecturas generativas para crear imágenes sintéticas y balancear el dataset:

Autoencoder Variacional (VAE)

Encoder convolucional → espacio latente (128-dim) → Decoder deconvolucional. Entrenado durante 50 épocas con función de pérdida combinada: MSE (reconstrucción) + KL divergence (regularización). Resolución: 64×64 px.

Conditional DCGAN (Deep Convolutional GAN)

Generador y discriminador condicionales por clase. El generador recibe ruido latente (100-dim) + embedding de clase y produce imágenes 64×64. Entrenado 100 épocas con Adam (lr = 2×10⁻⁴, β₁ = 0.5).

Balanceo del dataset

Se generaron 11,495 imágenes sintéticas con el DCGAN condicional para igualar todas las clases a 7,045 muestras:

LoRA (Low-Rank Adaptation)

Se aplicó LoRA/PEFT sobre el ViT-Base, inyectando adaptadores de bajo rango en las matrices de atención query y value:

Dropout0.1Regularización en adaptadores
Target modulesquery, valueMatrices de atención adaptadas
Params entrenables298K / 86M0.35% del total

Técnicas de optimización

  • Cuantización INT8: Cuantización dinámica post-entrenamiento de capas Linear.
  • Pruning L1 (30%): Poda no estructurada de pesos con menor magnitud.
  • Re-entrenamiento con sintéticos: LoRA re-entrenado con dataset aumentado (real + sintético).

Despliegue Gradio

Se implementó una interfaz web con Gradio que integra:

  • Clasificación con ViT + LoRA (merged) + diagnóstico descriptivo
  • Visualización Grad-CAM para explicabilidad
  • Modelo final guardado en formato Hugging Face con metadata de deploy

3.6 Etapa 6 — Mejoras Avanzadas

Detección multi-defecto con YOLO11

Se entrenó YOLO11n (nano) de Ultralytics para detección de objetos, permitiendo localizar y clasificar múltiples defectos simultáneamente con bounding boxes. YOLO11n ofrece ~19% menos parámetros que YOLOv8n (2.6M vs 3.2M) con mejor mAP.

Edge deployment

Se combinó pruning L1 (30%) + cuantización INT8 + exportación ONNX para obtener un modelo ultra-compacto compatible con dispositivos edge (drones, cámaras industriales).

Pipeline de inspección automatizada

Se integró un pipeline end-to-end que procesa lotes de imágenes, clasifica con ViT + GradCAM, aplica triage por severidad y genera reportes JSON estructurados para integración con sistemas CMMS/MRO.

6. Limitaciones y Trabajo Futuro

Limitaciones identificadas

  1. Resolución de imágenes sintéticas: Las imágenes generadas por la DCGAN (64×64 px) son de resolución inferior a los parches reales, lo que limita su contribución al entrenamiento. La mejora de +0.35 pp es modesta.
  2. Desbalance extremo en scratch: Aunque se generaron 5,994 imágenes sintéticas para esta clase, la calidad visual no es comparable a muestras reales — un generador de mayor resolución (StyleGAN2/3, Diffusion Models) podría ser más efectivo.
  3. Cuantización en CPU: La cuantización dinámica INT8 de PyTorch no acelera la inferencia en GPU y es ~5× más lenta en CPU respecto a GPU. Se requiere cuantización estática o TensorRT para aceleración real.
  4. Dataset específico: Los resultados están validados únicamente en datasets de Roboflow Universe. La generalización a imágenes capturadas en condiciones reales de campo (iluminación variable, ángulos diversos, contaminación superficial) requiere validación adicional.
  5. Clasificación vs. Detección: El pipeline actual ejecuta clasificación y detección de forma independiente. Una arquitectura unificada end-to-end sería más eficiente.

Trabajo futuro

  • Generación con Diffusion Models: Reemplazar la DCGAN por Stable Diffusion fine-tuned con ControlNet para generar parches de defectos de alta resolución (512×512) con control sobre la geometría del defecto.
  • YOLO26 NMS-free: Migrar a YOLO26 (última versión, enero 2026) que elimina la necesidad de Non-Maximum Suppression, reduciendo latencia y simplificando el pipeline de inferencia.
  • Segmentación semántica: Evolucionar de bounding boxes a máscaras pixel-wise para cuantificar el área exacta del defecto — información crítica para decisiones de mantenimiento.
  • Despliegue TensorRT: Compilar el modelo ViT para NVIDIA TensorRT con cuantización FP16/INT8 estática para inferencia <10 ms en GPUs edge (Jetson Orin).
  • Validación en campo: Evaluar el sistema con imágenes capturadas por drones de inspección en hangares reales para medir la brecha domain-shift.
  • Few-shot learning: Adaptar el sistema para clasificar nuevos tipos de defectos (corrosión, delaminación) con pocas muestras anotadas mediante meta-learning o prompt-tuning.

7. Conclusiones

Este proyecto demostró el desarrollo progresivo de un sistema de visión por computadora para la clasificación automatizada de defectos en superficies de aeronaves, avanzando desde un MLP clásico hasta un Vision Transformer con adaptación de bajo rango (LoRA).

Hallazgos principales

  1. La estructura espacial es fundamental (E1→E2, +41.35 pp): La transición de MLP a CNN demostró que preservar la relación espacial de los píxeles es crítico para el reconocimiento de patrones visuales. Los kernels convolucionales capturan bordes, texturas y formas que el MLP pierde al aplanar.
  2. El transfer learning es transformador (E2→E3, +5.37 pp): El conocimiento visual general aprendido en ImageNet (1.2M imágenes, 1000 clases) se transfiere eficientemente incluso a un dominio tan especializado como la inspección aeronáutica.
  3. LoRA demuestra la eficiencia de la adaptación de bajo rango (E3→E5, +1.85 pp): Con solo el 0.35% de parámetros entrenables (298K de 86M), ViT + LoRA supera al fine-tuning completo. La restricción de rango bajo actúa como regularización implícita que previene el sobreajuste.
  4. Los datos sintéticos aportan valor marginal pero estratégico (+0.35 pp): En un dataset ya abundante, el balanceo con imágenes generadas por DCGAN produce mejoras modestas pero consistentes, beneficiando especialmente a las clases sub-representadas.
  5. La optimización agresiva es viable (99.1% reducción): La cuantización INT8 reduce el modelo de 327 MB a 3 MB con solo −1.32 pp de penalización, demostrando que los modelos de deep learning contienen redundancia significativa explotable para despliegue en edge.

Contribución técnica

El sistema final integra clasificación con explicabilidad (ViT + LoRA + Grad-CAM), detección multi-defecto (YOLO11n), y un pipeline de inspección automatizada con triage por severidad — un sistema completo y funcional para soporte a la toma de decisiones en mantenimiento aeronáutico, desplegable como aplicación web o integrable con sistemas industriales existentes.
